1：安装svn  (在线安装)
    >yum -y install subversion
2：创建数据仓库目录
    >mkdir  /var/www/svndata
3：创建具体的目录
    >svnadmin create /var/www/svndata/game_test
    注：game_test是项目名字,及访问的根目录
4：配置简单的访问权限
    >cd /var/www/svndata/game_test/conf
    >vi svnserve.conf
    anon-access=none
    auth-access=write          
    password-db=passwd
    注：修改的文件前面不能有空格，否则启动svn server出错
    anon-access  控制非鉴权用户访问版本库的权限。取值范围为"write"、"read"和"none"。
                 即"write"为可读可写，"read"为只读，"none"表示无访问权限。
                 缺省值：read

    auth-access  控制鉴权用户访问版本库的权限。取值范围为"write"、"read"和"none"。
                 即"write"为可读可写，"read"为只读，"none"表示无访问权限。
                 缺省值：write

    password-db  指定用户名口令文件名。除非指定绝对路径，否则文件位置为相对conf
                 目录的相对路径。
                 缺省值：passwd

    authz-db     指定权限配置文件名，通过该文件可以实现以路径为基础的访问控制。
                 除非指定绝对路径，否则文件位置为相对conf目录的相对路径。
                 缺省值：authz

    realm        指定版本库的认证域，即在登录时提示的认证域名称。若两个版本库的
                 认证域相同，建议使用相同的用户名口令数据文件。
                 缺省值：一个UUID(Universal Unique IDentifier，全局唯一标示)
   >vi password
   xiewen=123456

5：svn服务启动与停止
    关闭：
>ps -ef|grep svnserve
[root@localhost /]# ps -ef|grep svnserve
root       2882      1  0 18:51 ?        00:00:00 svnserve -d -r /var/www/svndata/
root       2939   2848  0 19:14 pts/0    00:00:00 grep svnserve
>kill -9 2882
或者
>killall svnserve

   启动:
      >svnserve -d -r /var/www/svndata     
   注意：这个目录就是svn创建数据仓库目录,即我们第二步创建的目录

6：命令访问svn,window要svn命令必须安装时需要勾选client command
   >svn co svn://127.0.0.1/game_test

注：如果外部不能访问,检查一下防火墙    >service iptables status
 



7：如果实现特殊组合分组访问目录 ，
 将svnserve.conf里面注释authz-db=authz
>vi authz  进行如下修改:
 
[groups]
 admin=xiewen
[njlrxx:/]
xiwen = rw
@admin=rw
*=r

注：
admin   ======   分配一个admin用户组，成员有xiewen
[njlrxx:/]   =====   svn项目名，项目的根目录
@admin=rw  ==   admin组成员拥有读写权限
*=r        ======   则表示“那些人只能读，不能写”
